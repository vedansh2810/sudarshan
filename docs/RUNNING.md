# Running Sudarshan

This document explains how to run the project locally and common troubleshooting steps.

## Prerequisites

- Python 3.10+ (3.12+ recommended)
- Node.js 18+ (optional - only needed if modifying templates)
- (Optional) PostgreSQL or Supabase - the app falls back to SQLite when PostgreSQL is not available
- (Optional) Redis for Celery task queue

## Quick Start (Recommended)

The fastest way to set up and run the project on any system:

```bash
python start.py
```

This single command handles everything:
1. Creates a virtual environment (`.venv/`)
2. Installs Python dependencies from `requirements.txt`
3. Copies `.env.example` to `.env` and auto-generates a secure `SECRET_KEY`
4. Installs npm packages and builds Tailwind CSS (if npm is available)
5. Creates required data directories
6. Starts the Flask server at `http://localhost:5000`

On subsequent runs, it detects the setup is already complete and starts the server immediately.

### Other start.py Commands

```bash
python start.py --check    # Show setup status without running
python start.py --setup    # Force re-run setup even if already complete
python start.py --run      # Skip setup checks, just start the server
```

## Manual Setup (Alternative)

If you prefer to set up manually:

1. Create and activate a virtual environment:

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate
```

2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Copy the example env and edit values:

```bash
cp .env.example .env
# Edit .env and set your Supabase credentials (required for login)
# See .env.example for detailed instructions
```

4. Build Tailwind CSS (optional, requires npm):

```bash
npm install
npm run build:css
```

5. Run the dev server:

```bash
python run.py
```

## Configuration

Edit `.env` to configure:
- **SECRET_KEY** - Flask session secret (auto-generated by `start.py`)
- **SUPABASE_URL / SUPABASE_ANON_KEY / SUPABASE_SERVICE_KEY** - Required for login/register
- **DATABASE_URL** - PostgreSQL URI (falls back to SQLite if not set)
- **GROQ_API_KEY** - Groq API key for AI features (optional)
- **REDIS_URL** - Redis URI for Celery task queue (optional)

See `.env.example` for the full list with setup instructions.

## Notes and Troubleshooting

- **Missing env vars:**
  - In development the app will warn about missing env vars (e.g., `SUPABASE_URL`) and continue using the debug fallback for authentication.
  - In production the app will raise an error if critical vars like `SECRET_KEY` are missing.

- **Login/Register not working:**
  - Ensure `SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_KEY` are set correctly in `.env`.
  - Add `http://localhost:5000/auth/callback-handler` to your Supabase project's redirect URLs (Authentication > URL Configuration).

- **TLS verification when scanning targets:**
  - For local testing of intentionally broken TLS targets, set `ALLOW_INSECURE_TARGETS=1` in your `.env`. Do NOT enable this in production.

- **Running tests:**

```bash
pytest tests/ -v
```

- **Running Celery worker (optional):**

```bash
celery -A app.celery_app:celery worker --loglevel=info
```

- **Rebuilding Tailwind CSS (after template changes):**

```bash
npm run build:css       # One-time build
npm run watch:css       # Watch mode (auto-rebuild)
```

If something still fails, check `logs/` for application logs and open an issue with the failing traceback and the contents of your `.env` (omit secrets).
